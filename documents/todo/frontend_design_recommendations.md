# Frontend Design Recommendations - Premium Polish Round 2

**Date:** 2026-06-12
**Scope:** Marketing site (`demo/` Next.js app) - all pages and shared chrome
**Status:** Suggestions only - no code changes made
**Constraints:** Font + theme colours stay as-is. All recommendations must be performant and align with `vercel-react-best-practices` (CSS transform/opacity-first, no new scroll listeners, `next/dynamic` + conditional loading for any JS).

---

## Audit Summary

Surveyed every page (home, pricing, about, contact, 404, privacy), the shared chrome (navbar, footer, mobile menu), the button/badge/form elements, and the full animation guide (`documents/guides/_animations.md`, all 53 sections). Nothing below duplicates the 40+ effects already shipped (aurora, orbs, spotlight, magnetic CTAs, luminance sweep, prismatic entrances, border beam, hue shift, etc.).

The home and pricing pages are already at a premium tier, so the biggest "wow per effort" now lives in the **neglected surfaces**: the footer (links have literally **no hover state**), the about page (none of the premium systems are enabled there), the mobile menu (the only unanimated interaction left), the contact form (the conversion endpoint for every "Book a demo" CTA), and global fit-and-finish (no `::selection`, scrollbar, or caret theming exists - verified via grep). The most differentiating single idea is making the hero *demonstrate* the product instead of describing it (Rec 9).

### Key Evidence Found

- ❌ `FooterLink` (`demo/src/components/sections/footer-with-newsletter-form-categories-and-social-icons.tsx:18`) renders links with zero hover styling - the flattest surface on the site.
- ❌ `demo/src/app/404/page.tsx` is the only page with no animation or atmosphere at all.
- ❌ `demo/src/app/about/page.tsx` never passes `stickyEyebrow`/`sectionHue`/`ScrollHighlight`, and `TeamMember` cards have no hover treatment.
- ❌ No `::selection`, `scrollbar-color`, or `caret-color` anywhere in `demo/src/app/globals.css`.
- ❌ No native CSS scroll-driven animations (`animation-timeline`) used anywhere yet.
- ✅ Plan-comparison table already has sticky glass headers - that candidate was dropped after verification.

## Top 10 Recommendations

| ID | Name | Description |
| --- | --- | --- |
| 1 | Monumental Footer Wordmark | The site footer (the tinted band containing the "Product / Support / Legal" link columns and the fineprint "© 2026 Bulma Pty Ltd", rendered by `FooterWithNewsletterFormCategoriesAndSocialIcons`) is currently the flattest surface on the site, and its links ("Pricing", "Contact", "Privacy Policy") have no hover state at all. Add a viewport-spanning "BULMA" wordmark (~14vw, the existing Mona Sans display font) cropped by the footer's bottom edge, filled with a vertical mist-tone gradient via `background-clip: text` that fades to transparent, with a single slow luminance pan (reusing the existing `luminance-sweep` pattern) triggered once by the existing `useScrollAnimation` hook. Pair it with 200ms `color` + 2px `translate-x` hover transitions on the footer links. This is the editorial signature move of premium product sites (Vercel, Linear class) and gives every page a memorable closing frame. Performance: static JSX (`rendering-hoist-jsx`), pure CSS gradient, one IntersectionObserver, zero new scroll listeners. |
| 2 | 404 "Lost Policy" Moment | The 404 page (`demo/src/app/404/page.tsx`, headline "Page not found" with the "Go back home" link) is the only page with zero animation or background atmosphere. Place an oversized ghost "404" numeral behind the headline using `-webkit-text-stroke` outline text at ~6% opacity, and reuse the existing `DotMatrix` canvas (cursor-proximity dots, currently only on the stats section) plus `CursorSpotlight` for ambience. A dead-end becomes a brand moment that makes people *want* to screenshot your error page. Performance: both components are already RAF-throttled and retina-aware; load them with `next/dynamic` since the 404 route is rarely hit (`bundle-dynamic-imports`), so the main bundle is untouched. |
| 3 | About Page Parity Pass | The about page (hero "Built by brokers, for brokers.", the "Our team" grid, and its stats section) sits a full polish tier below home/pricing: it never passes the existing `stickyEyebrow`/`sectionHue` props, has no `ScrollHighlight` phrases in its subheadlines, and the `TeamMember` photo cards (Leslie Alexander, Michael Foster, etc.) are static rectangles with only a fade-up entrance. Enable those existing props, then give team cards a hover treatment: photos rest at `filter: saturate(0.7)` and bloom to full colour with a gentle 1.04 Ken Burns `scale` inside the existing `overflow-hidden` frame, while the card lifts (`-translate-y-1` + soft shadow) and an accent underline grows beneath the name. This makes the human page feel alive and consistent with the rest of the site's "sleek" tier. Performance: hover effects are GPU-composited `transform`/`filter` only; the section props already exist, so this adds almost no new JavaScript. |
| 4 | Contact Form Submit Choreography | The "Send a message" card on /contact (fields "Name", "Work email", "How can we help?", and the "Send message" button in `contact-form.tsx`) currently swaps button text to "Sending..." and shows a plain green alert on success - underwhelming for the page every "Book a demo" and "See it in action" CTA funnels into. Add a `focus-within` glow on the card's existing ring, labels that tint to full contrast when their field focuses (pure CSS `:has()`/peer selectors), and a submit morph: the button width eases into a pill-circle with a conic-gradient spinner mask, then resolves into the existing `AnimatedCheckmarkIcon` stroke-draw, with the success panel entering on the existing `faq-spring` overshoot easing. The single conversion action on the site will feel engineered rather than templated. Performance: all state-driven class toggles on existing React state, no new dependencies, no re-render hot paths (`rerender-functional-setstate` already in place). |
| 5 | Mobile Menu Entrance Choreography | The full-screen mobile menu (the `dialog#mobile-menu` opened by the hamburger button in the navbar, listing "Pricing", "Contact", "Log in") currently snaps in instantly as a solid sheet - the last unanimated interaction on the site, and the one every mobile visitor touches. Make the panel a glass surface (`bg-mist-100/90` + `backdrop-blur-xl`, matching the scrolled navbar) that fades in over 200ms, with links rising via a CSS-only keyframe and 60ms staggered `animation-delay` applied through the native `dialog[open]` selector, and the close icon rotating 90° on entry. This delivers polish parity with the desktop glassmorphism navbar where mobile traffic actually lives. Performance: triggered purely by the native dialog open state - no JS timeline, no listeners, zero bundle growth. |
| 6 | Native Scroll-Driven Screenshot Depth | The two feature screenshots in the "Everything you need to navigate lender policies with confidence." section ("Policy Q&A" and "Lender Comparison" `Screenshot` frames) animate once on entrance, then sit inert for the rest of the scroll. Add CSS scroll-driven animations (`animation-timeline: view()`) so each frame continuously settles - `scale` 0.97 → 1.0 with a softening `box-shadow` - as it traverses the middle band of the viewport, wrapped in `@supports (animation-timeline: view())` so other browsers keep today's behaviour. The page gains a continuous physical depth that entrance-only animation cannot provide, at 3% scale so it reads as weight, not parallax gimmickry. Performance: this is the gold-standard primitive - it runs entirely on the compositor with zero JavaScript, no IntersectionObserver, and no scroll listeners. |
| 7 | Global Fit-and-Finish Pack | Site-wide details in `globals.css` that are currently browser defaults: selecting the hero headline "Your AI assistant for policy questions." shows the stock blue highlight, the scrollbar is unthemed, and form carets are default - instant "template" tells on an otherwise bespoke site. Define a branded `::selection` (mist-300 background with mist-950 text in light mode, inverted in dark), a slim `scrollbar-color` pairing on the mist palette, `caret-color` for the contact form fields, plus `text-wrap: balance` on `Heading` and `text-wrap: pretty` on body `Text` to eliminate orphan words in headlines like "Simple pricing for every brokerage." These are the invisible details that make visitors *feel* craftsmanship without identifying why. Performance: pure CSS declarations with zero runtime or layout cost. |
| 8 | Announcement Badge Live Pulse | The announcement pill above the hero headline reading "Now covering all major Australian lenders - See the list" (`AnnouncementBadge`) is a static tinted pill with only a background hover change. Prepend a 6px accent status dot with a soft expanding halo ring, reusing the existing `presence-pulse` keyframe (2.5s cycle, `scale` 1 → 1.15 with opacity fade) currently applied to testimonial avatars. A breathing "live" indicator turns the coverage claim into something that reads as current and monitored - which is literally the product promise ("policies updated regularly"). Performance: one extra span plus an existing keyframe, animating `transform`/`opacity` only. |
| 9 | "Ask Bulma" Sample Question Chips | Directly beneath the hero CTA row ("Try Bulma free" / "See it in action"), visitors currently only *read about* the Q&A product while looking at a static screenshot. Add a row of three glass question chips using real questions from the FAQ copy ("What lenders go to 95% LVR?", "What add-backs does CBA accept?", "Compare the big 4's LMI requirements") - selecting one blur-fades (the existing 400ms `filter: blur()` + `scale` language from `BlurTransitionText`, deliberately not a typewriter) a compact pre-baked answer card over the hero screenshot frame, including the source-attribution footer that is Bulma's key differentiator. This is the highest-differentiation move available: the hero converts from claim to demonstration. Performance: chips render statically for SEO; the answer-card module loads via `next/dynamic` only on first interaction (`bundle-conditional`) and preloads on chip hover (`bundle-preload`), leaving the initial bundle untouched. |
| 10 | CTA Bookend Atmosphere | The closing CTA sections - "Ready to spend less time on policy research?" on home (`CallToActionSimple`) and "Have more questions?" on pricing (`CallToActionSimpleCentered`) - sit on a flat background, so the atmospheric energy of the hero (aurora, orbs, spotlight) dies exactly where you ask for the conversion. Reuse the existing `AuroraBackground` at reduced opacity (~60%, two layers instead of three) behind these sections, creating a visual bookend that mirrors the hero and subconsciously signals "the journey ends where it began - act now". Performance: the component already pauses its keyframes off-screen; mount it conditionally from the section's existing `useScrollAnimation` visibility so it only loads near the viewport (`bundle-conditional`), and the sections already use `content-visibility` deferral. |
